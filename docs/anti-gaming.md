# Anti-Gaming

Buildmarks should be useful because it refuses easy vanity metrics.

## Rules

1. Raw commit count is not a primary score.
2. Contribution streak is not scored.
3. Follower count is not scored.
4. Language percentages are not treated as engineering quality.
5. Stars and forks are context, not default profile-card tier dimensions.
6. Forked repositories are excluded by default.
7. Archived repositories are excluded by default.
8. One popular repository must not dominate the whole profile.
9. Low public activity must never produce a harsh personal label.
10. Every score must expose evidence and limitations.
11. Generated cards and reports must disclose their data scope.

## Labels

Labels must describe visible public evidence, not personal worth.

Use labels such as:

- Maintainer-Builder
- Productized Builder
- Builder
- Steady Shipper
- Well-Documented Project
- General Signal Profile

Avoid labels such as:

- elite developer
- senior engineer
- hireable
- top percentage
- S class
- genius

Buildmarks should make the evidence clearer, not make a person smaller.
